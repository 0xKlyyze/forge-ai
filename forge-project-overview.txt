# Project Overview: AI Development Workflow Manager

## Core Concept

A centralized cloud-based project management application designed specifically for AI-powered development workflows. The app eliminates the friction of managing multiple project files across devices by providing an auto-organized, template-based system for storing, editing, and previewing all project-related artifacts in one place.

**Primary Value Proposition:** Instant project setup with enforced structure + seamless cross-device access + unified editing environment for documentation, mockups, and assets.

## Target User

**Profile:** Solo developer (18 years old) building AI-powered applications using AI development agents. Works across multiple devices and manages several projects simultaneously. Needs quick context switching and minimal organizational overhead.

**Current Pain Points:**
- File hunting across devices when resuming paused projects
- Manual folder structure creation for each new project
- Context loss when switching between projects
- No unified preview environment for UI mockups
- Time wasted on file management instead of building

**Success Metrics:** Reduces project setup time to under 30 seconds. Eliminates need for local file searching. Provides instant context when reopening old projects.

## Project Scope

- **Certainty Level:** 3/5 (Moderately committed, exploring viability)
- **Build Scope:** 2/5 (Lean personal tool, not enterprise-grade)
- **Commercial Intent:** 1/5 (Purely personal use)

**Implication:** Focus on MVP functionality. Prioritize speed of development over scalability. Use simple, proven tech stack. Avoid over-engineering.

## Technical Architecture

### Frontend Stack
- **Framework:** React with TypeScript
- **State Management:** React Context or Zustand (lightweight)
- **Routing:** React Router
- **Styling:** Tailwind CSS (rapid development)
- **Code Editor:** Monaco Editor (VSCode engine)
- **Mockup Preview:** CodeSandbox Sandpack React component
- **Markdown Editor:** React Markdown Editor library or build custom

### Backend & Storage
- **Database:** Firebase Firestore or Supabase (PostgreSQL)
  - Store project metadata, file references, task data
- **File Storage:** Firebase Storage or Supabase Storage
  - Store actual file contents (markdown, code, images)
- **Authentication:** Firebase Auth or Supabase Auth
  - Simple email/password or Google OAuth
  - Enables multi-device access

**Project:**

## Critical Implementation Notes

### Source of Truth Priority (Phase 2)
- This feature prepares the app for future API/MCP integration
- When AI agents query project info, they should respect priority levels
- UI should make priority visible but not overwhelming
- Default priority: 5 (mid-level)
- User can adjust per file as needed

### CodeSandbox Integration
- Use Sandpack component (official React wrapper)
- Configure for React environment
- Handle dependencies gracefully (default to React only for MVP)
- Error boundaries for preview failures

### Auto-Save Strategy
- Debounce editor changes (2-3 second delay)
- Visual indicator when saving (subtle)
- Prevent data loss on network issues (local cache)

### Performance Considerations
- Lazy load file contents (don't load all files at once)
- Paginate project list if >50 projects
- Optimize image uploads (compress before storage)
- Code splitting for route-based loading

## Risk Mitigation

**Potential Issues:**
1. **Sandpack Performance:** May be slow for complex components
   - Mitigation: Start with simple React components only
2. **Storage Costs:** Image/file storage could add up
   - Mitigation: Firebase free tier is generous; compression helps
3. **Sync Conflicts:** Rare if only one user, but possible
   - Mitigation: Last-write-wins for MVP (version history later)

## API/MCP Future Considerations

The "Source of Truth Priority" feature enables future AI agent integrations:
- AI agents can query project docs via API
- Priority system ensures correct info retrieval
- Potential endpoints:
  - `GET /project/:id/docs` (returns all docs with priority)
  - `GET /project/:id/context` (returns consolidated project info)
  - `POST /project/:id/task` (create task from external agent)

This positions the app as a knowledge base for AI development workflows.
